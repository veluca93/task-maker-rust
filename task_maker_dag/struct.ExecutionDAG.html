Struct task_maker_dag::ExecutionDAG

source · 
pub struct ExecutionDAG {
    pub data: ExecutionDAGData,
    pub callbacks: Option<ExecutionDAGCallbacks>,
}
Expand description

A computation DAG, this is not serializable because it contains the callbacks of the client.

Fields§

§data: ExecutionDAGData

Serializable part of the DAG with all the executions and files.

§callbacks: Option<ExecutionDAGCallbacks>

The list of callbacks for the items of the DAG.

Upon cloning this DAG, the callbacks won’t be available anymore. This is an Option to check that the callbacks are never accessed on the clones.

pub fn write_file_to<F: Into<FileUuid>, P: Into<PathBuf>>( &mut self, file: F, path: P, executable: bool )

When file is ready it will be written to path. The file must be present in the dag before the evaluation starts.

If the config dry_run is set to true the calls to this function are no-op.

If the generation of the file fails (i.e. the Execution that produced that file was unsuccessful) the file is not written.

pub fn get_file_content<G: Into<FileUuid>, F>( &mut self, file: G, limit: usize, callback: F )where F: FnOnce(Vec<u8>) -> Result<(), Error> + 'static,

Call callback with the first limit bytes of the file when it’s ready. The file must be present in the DAG before the evaluation starts.

If the generation of the file fails (i.e. the Execution that produced that file was unsuccessful) the callback is called anyways with the content of the file, if any.

Calling this method twice on the same file will panic (this behaviour can be changed in the future).
